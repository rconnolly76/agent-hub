# Top 5 Recommendations Contract (v1.0)

This contract standardizes the priority recommendations generated by every skill run.

## File name and placement

- Report skills: `_top-5-recommendations.json` in repo root
- Content-bundle skills: `_top-5-recommendations.json` inside the bundle directory (for example `docs/_top-5-recommendations.json`)

The push script automatically uploads this payload as `topRecommendations`.  
If missing, it generates one from available report content.

## Schema

```json
{
  "version": "1.0",
  "skillType": "ux-journey-reviewer",
  "generatedAt": "2026-04-15T10:00:00.000Z",
  "recommendations": [
    {
      "priority": "P1",
      "title": "Resolve critical navigation failures",
      "action": "Fix broken nav states on run detail and verify with rerun.",
      "rationale": "Critical path failure blocks user progress.",
      "severity": "critical"
    }
  ]
}
```

## Rules

- `version` must be `"1.0"`.
- `recommendations` must contain 1-5 entries (ingest stores first five ordered by priority).
- Priorities are `P1` to `P5`.
- Required per recommendation:
  - `priority`
  - `title`
  - `action`
- Optional:
  - `rationale`
  - `severity` (`critical|warning|info|low|investigate`)

## Ingest behavior

- If payload is provided and valid, it is stored in `runs.rawMetadata.topRecommendations`.
- If missing, ingest derives top recommendations from findings/metrics.
- Executive summary normalization uses these recommendations first, and always appends a prioritized top-5 block.
